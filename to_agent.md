= 功能规划

项目核心流程分为“采集-答题-批改-强化”四个阶段：

- *多模态题目采集*：支持拍照上传纸质题目，通过 OCR 或多模态大模型提取题干，自动识别并渲染数学公式（LaTeX）,（或者直接文本输入大致题意）。
- *灵活答题系统*：
  - 用户可在 Web 端直接输入答案。
  - *大题支持图片上传*：针对复杂数理大题，用户可直接拍摄手写解题过程，由 AI 进行图像识别与步骤解析。
- *深度智能批改*：
  - *多维度评价*：不仅判断对错，还包括步骤分、错因分析（如计算失误、公式记忆偏差、逻辑断层）以及改进建议。
- *错题管理与举一反三*：
  - *明文存储*：错题以 JSON 格式存储，方便与 AI 进行上下文交互及用户自行查阅以及导出错题。
  - *变式训练*：针对错题，AI 自动生成 3 道知识点相同但参数或背景不同的变式题。


= 技术栈分析

项目采用前后端分离架构，核心依赖于多模态大模型（LMM）：

- *前端*：React.js / Vue 3 + Tailwind CSS。使用 *KaTeX* 处理数理公式的高质量渲染。
- *后端*：Python 。负责处理 AI 请求及文件系统交互。
- *AI 模型层*：
  - *识别*：初期并行测试“通用 OCR API”与“GPT-4o / Qwen-VL”方案。
  - *推理*：使用 Claude 3.5 Sonnet 或 DeepSeek-V3 负责数理逻辑推导。
  - *成本控制*：采用“轻任务优先走 OCR/规则链路，复杂任务再调用大模型”的分层策略，并通过上下文裁剪、结果结构化输出与阶段性缓存控制 token 消耗。
- *存储方案*：*明文文件存储*（Local File System）。以 JSON 格式保存用户数据，确保存储的透明性与易处理性。
